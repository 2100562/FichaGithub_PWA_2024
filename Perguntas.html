  1. O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?

    A operação de commit permite guardar um conjunto local de alterações ao código enquanto que a operação de push submete ou envia os commits locais para um repositório centralizado. A operação de pop está relacionada com a stash que é um lugar temporário para guardar alterações que ainda não foram guardadas com o commit, permitindo alterar entre branches sem transpor essas alterações entre branches.

  2. Quais são as vantagens de usar Branchs num repositório git?

    Utilizar branches permite que diversos engenheiros contribuam em simultâneo para o mesmo código, existindo no final um merge de todas as alterações num branch final. Por outro lado, diversas técnicas de organização da evolução do código, utilizam branches para representar uma nova Sprint, ou uma nova funcionalidade, ou até um bug-fix em produção.

  3. Nos Merges de conteúdos resultantes de um “Pop” podem resultar em conflitos. Em que consistem e como podem ser resolvidos esses conflitos?

    Os conflitos de merge resultam quando existem múltiplas alterações à mesma linha de um ficheiro. O git apenas consegue resolver automaticamente conflitos em linhas distintas. Para resolver os conflitos existem duas opções, ou escolher uma das alterações como a final ou manualmente unir as alterações para produzir um ficheiro final que contenha todas as alterações.

  4. “Clone” e “Fork” são formas diferentes de replicar um repositório Git. Quais as diferenças entre eles?

    Tanto Clone como Fork permitem replicar um repositório no entanto com algumas diferenças. Por um lado, o Clone, apenas irá replicar o repositório central como Github ou Gitlab para o computador onde se executa clone, mantendo o remote original, permitindo colaborar e submeter alterações para esse repositório central, assumindo que o utilizador tem permissões. Por outro lado, o Fork replica o repositório central de uma conta para outra conta, permitindo assim que utilizadores sem permissões possam fazer alterações "locais" e podendo através de PR submeter essas alterações para o repositório original.
